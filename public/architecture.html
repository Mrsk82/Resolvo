Technical Architecture

System components, request flow, and infrastructure overview

🏗 System Architecture

🌐 Client Layer

  • Single Page App (index.html ~1.4MB)
  • Vanilla JS — no framework
  • CSS design system (design tokens)
  • Progressive Web App (manifest.json)
  • GA4 analytics tracking
  • Session token in localStorage

⚙️ Server Layer

  • Node.js + Express (~5000 lines)
  • Single process, multi-tenant routing
  • Session tokens in memory (Map)
  • Password reset tokens in owner.json
  • Rate limiting: 5 attempts/15min/IP
  • Background jobs via node-cron

🗄 Data Layer

  • JSON file database (no SQL)
  • data/owner.json — platform owner
  • data/brands/:slug/db.json — per brand
  • Complete isolation between brands
  • Atomic writes (JSON.stringify)
  • Backup via /api/backup endpoint

📧 Email Layer

  • Nodemailer via Gmail SMTP
  • Owner email: contact@resolvogroup.com
  • Per-brand SMTP if configured
  • Falls back to owner SMTP
  • IMAP polling for ticket creation
  • Cron: reminders, nurture, digest

🔐 Auth Layer

  • bcrypt cost 10 for all passwords
  • UUID v4 session tokens
  • Owner vs Brand session types
  • Feature flags resolved at login
  • ownerOnly middleware for owner routes
  • Brand slug from session (no spoofing)

🚀 Deployment

  • Hostinger VPS (Ubuntu)
  • PM2 process manager
  • GitHub → manual git pull deploys
  • Nginx reverse proxy (optional)
  • Base URL: app.resolvogroup.com
  • Local: PORT=3000
